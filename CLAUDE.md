## กฎเหล็ก — ละเมิดไม่ได้

### iOS / HTML viewer (เคยพังมาแล้วทุกข้อ)
- ❌ `confirm()` / `alert()` — แอปบล็อก กดแล้วเงียบ → ใช้ **แตะสองครั้งยืนยัน**
- ❌ `location.reload()` — บางแอปบล็อก → ล้าง DOM แล้ว render ใหม่เอง
- ❌ `beforeunload` — iOS ไม่ยิงเสมอ → เซฟทุกวินาที
- ❌ **ห้ามลิงก์ฟอนต์/ไลบรารีจากเน็ต** — ต้องเปิดออฟไลน์ได้
- ⚠️ Safari ลบ localStorage หลัง 7 วันถ้าไม่ได้ติดตั้งเป็น PWA → เตือนให้ "เพิ่มไปยังหน้าจอโฮม"

### ภาษาและน้ำเสียง
- ❌ **ห้ามอีโมจิใน UI** — ใช้ `ART.*()` จาก `art.js` แทน
- ❌ **ห้ามคำเชียร์** ("เยี่ยมมาก" "สู้ ๆ" "เก่งมาก")
- ✅ ทุกข้อความสรุปผลจบด้วย **สิ่งที่ต้องทำต่อ** ไม่ใช่คำตัดสิน
  - เช่น "ผิดทั้งที่มั่นใจ — ข้อ 7, 12 / เข้าใจคลาดเคลื่อน อ่านเฉลยให้จบก่อนทำข้ออื่น"
  - แม้ได้เต็ม: "ถูกทั้งชุด 15/15 / ไม่มีข้อค้าง ชุดถัดไปจะเพิ่มสัดส่วนข้อลึก"

### โค้ด
- ⚠️ `el.style.cssText = '...'` **ล้างสไตล์เดิมทั้งหมด** — อย่าเซ็ต `style.xxx` ทีละตัวก่อนหน้า (เคยทำให้ flex หาย รูปลอย)
- ⚠️ **ตัวคั่นใช้ `•` เท่านั้น ห้ามใช้ `·`** (ดู THEME.md บั๊ก 3)
- ⚠️ `<input>` ตัดเนื้อหาตามกล่อง (ต่างจาก `<div>`) — ระวังสระบนโดนเฉือน

## สเปกเอนจินทำโจทย์ (`/posn/set.html`)

ทับทิมเป็น **visual learner** · **หลีกเลี่ยงเมื่อเจอโจทย์ยาก** → ต้องมีทางออกเสมอ
· **มีปัญหากับ word problem** → ต้องมีปุ่มคำแปลโจทย์

ต่อข้อ:
- ป้ายชั้น (🟢 ปูพื้น / 🟡 ระดับข้อสอบ / 🔴 ท้าทาย) + `★ ข้อลึก` ถ้ามี
- ปุ่ม **คำใบ้** (`ART.bulb`) → เผยคำแปลโจทย์ (`ART.note`) ถ้าข้อนั้นมีฟิลด์ `tr`
- ตัวเลือก 4 ข้อ **หรือ** ช่องเติมคำตอบถ้า `type:"fill"`
- ปุ่มความมั่นใจ **ก่อนเฉลย**: มั่นใจ (`ART.sure`) / ไม่มั่นใจ (`ART.unsure`)
- ปุ่ม **เฉลย** · **ล้าง** · **ไม่รู้เลย**
- เฉลยต้องมีขั้น 1→2→3 + ตารางไล่ค่า (รองรับ `<pre>` `<table>` `<svg>` ใน HTML)

จับเวลา:
- **นับขึ้น** (ไม่ใช่นับถอยหลัง) + มีเป้าหมาย `targetMin`
- **หยุดเองเมื่อสลับแอป** (`visibilitychange` + `pagehide` + `blur`)
- **กันเวลากระโดด:** ถ้าช่องว่างระหว่าง tick > 5 วินาที = แอปถูกพักหลังบ้าน → ไม่นับ + หยุดอัตโนมัติ

สรุปผลท้ายชุด — 4 กลุ่ม (ดูถ้อยคำจริงใน `theme.html`):
| กลุ่ม | ไอคอน | สิ่งที่ต้องทำต่อ |
|---|---|---|
| ผิดทั้งที่มั่นใจ | `alert` | เข้าใจคลาดเคลื่อน อ่านเฉลยให้จบ |
| ถูกแต่ไม่มั่นใจ | `unsure` | ยังไม่แน่น ทำซ้ำภายใน 3 วัน |
| ไม่รู้เลย | `book` | เปิดเล่ม บทที่เกี่ยว |
| พลาด | `grid` | เขียนตารางไล่ค่าใหม่ |

localStorage: ตรวจ **schema + จำนวนข้อ** ก่อนใช้ ไม่งั้นค่าชุดเก่าลากมาปน · เซฟ 2 ที่ (localStorage + IndexedDB)

## สคีมาข้อมูล

`data/posn/index.json` — **สร้างแล้ว 6 ชุด · 90 ข้อ** (ดึงจากชุด HTML เดิม ตรวจผ่าน 0 ข้อผิดพลาด)
```json
{ "room":"posn-com",
  "sets":[{ "id":"260715-math-count", "date":"260715",
            "subject":"math",              // math | com | mock
            "title":"เลข — การนับ",
            "n":10, "targetMin":23, "week":3 }] }
```

`data/posn/<id>.json`
```json
{ "id":"260715-math-count", "n":10, "targetMin":23,
  "questions":[{
    "q":"โจทย์ (HTML — ฝัง <b> <pre> <table> <svg> ได้)",   // บังคับ
    "c":"1,260",                                            // บังคับ — เฉลย
    "ex":"คำอธิบาย (HTML) ขั้น 1→2→3 + ตารางไล่ค่า",         // บังคับ
    "ch":["1,260","5,040","2,520","630"],   // ไม่มี = อัตนัย (ต้องมี type:"fill")
    "tr":"คำแปลโจทย์",                       // ไม่บังคับ — ปุ่มคำใบ้
    "layer":1,                              // 1=ปูพื้น 2=ระดับข้อสอบ 3=ท้าทาย
    "topic":"permutation",                  // หัวข้อย่อย
    "sec":"เลข",                            // ใช้ใน mock (แบ่งส่วน)
    "deep":true,                            // ★ ข้อลึก
    "type":"fill"                           // อัตนัย
  }] }
```

**ข้อควรระวังจากข้อมูลจริง:**
- `layer` **ไม่มีในชุด mock** (ใช้ `sec` + `topic` แทน) → อย่า assume ว่ามีเสมอ
- `ch` ไม่มี 13 ข้อ (อัตนัย) → เช็ค `type==="fill"`
- `q` และ `ex` เป็น **HTML** → ต้อง `innerHTML` ไม่ใช่ `textContent`

เป้าเวลา: เลข **2.3 นาที/ข้อ** · คอม **2.7 นาที/ข้อ** · mock **1.5 นาที/ข้อ**
